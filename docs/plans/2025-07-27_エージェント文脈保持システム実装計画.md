# 2025-07-27 エージェント文脈保持システム実装計画

## 概要

複数のエージェント間での文脈保持とセッション管理を実現するための包括的なソリューションを実装します。現在発生している「確認フロー後のコンテキスト喪失問題」を解決し、スムーズな対話継続を可能にします。

## 問題分析

### 根本原因
1. **セッション管理の不備**: LINE Webhook毎に新しいエージェントインスタンスが生成される
2. **状態永続化の欠如**: 確認データ等の中間状態が保存されない
3. **ユーザー識別の問題**: 会話スレッドとユーザーの紐付けが不十分
4. **エージェント間の情報共有不足**: 各エージェントが独立して動作している

### 影響範囲
- WorkLogRegistrationAgent（作業記録登録）
- FieldAgent（圃場管理）
- SearchAgent（検索機能）
- その他の確認フローを持つ全エージェント

## 解決アプローチ

### 1. セッション管理レイヤーの導入

Redis + MongoDB ハイブリッド方式を採用し、以下の役割分担で実装：

- **Redis**: 短期セッション（1時間）、リアルタイム状態管理
- **MongoDB**: 長期会話履歴（30日）、ユーザープロファイル

### 2. 文脈保持アーキテクチャ

```
┌─────────────────┐
│   LINE Webhook  │
└─────────┬───────┘
          │
┌─────────▼───────┐
│ SessionManager  │ ← 新規コンポーネント
│ - Redis Cache   │
│ - User Context  │
└─────────┬───────┘
          │
┌─────────▼───────┐
│ ContextualAgent │ ← 拡張コンポーネント
│ - Memory Access │
│ - State Resume  │
└─────────┬───────┘
          │
┌─────────▼───────┐
│ Strategy Layer  │ ← 既存活用
│ - Registration  │
│ - Confirmation  │
└─────────────────┘
```
